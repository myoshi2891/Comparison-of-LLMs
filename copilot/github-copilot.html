01

GitHub Copilotとは？

AIを活用したコーディングアシスタントの全体像を理解しましょう

🤖
AIペアプログラマー
GitHub Copilotは、コードを書く際にリアルタイムで提案を行うAIアシスタントです。行全体や関数全体の補完、コードの説明、バグ修正まで幅広くサポートします。
⚡
55%の生産性向上
GitHub社の調査では、Copilot利用者はコード作成速度が最大55%向上し、開発者満足度が75%向上したと報告されています。
🌐
あらゆる環境で利用可能
VS Code、JetBrains、Xcode、Neovim、Visual Studio、Eclipse、Azure Data Studio、さらにターミナル（CLI）まで対応しています。
🔗
GitHub完全統合（2026年）
2026年時点では、コードレビューエージェント、MCPサポート、Copilot Spaces、エージェントモードなど強力な機能が追加されています。

03

ステップバイステップセットアップ

VS Codeを例にした導入手順です。10分で完了できます

01
GitHubアカウントの準備
github.com にアクセスしてアカウントを作成（既存アカウントでも可）。 プランを選択します。初めての方はCopilot Freeからスタートがおすすめです。
  • github.com → Settings → Copilot → 有効化
  • 学生の方はGitHub Student Developer Pack（無料Pro）を申請
  • Proプランの30日間無料トライアルも活用可能
02
VS Code拡張機能のインストール
VS Codeを開き、拡張機能マーケットプレイスから「GitHub Copilot」を検索してインストールします。
TERMINAL 
# VS Code コマンドパレットから実行（Ctrl/Cmd + P）
ext install GitHub.copilot
ext install GitHub.copilot-chat

# または コマンドラインから
code --install-extension GitHub.copilot
code --install-extension GitHub.copilot-chat
03
GitHubアカウントでサインイン
VS Code左下のアカウントアイコン、またはコマンドパレットから「GitHub Copilot: Sign in to GitHub」を実行。ブラウザでOAuth認証を完了します。
  • Ctrl/Cmd + Shift + P → "GitHub Copilot: Sign in"
  • ブラウザが開くのでGitHubアカウントで承認
  • 認証後、エディタに戻ると右下にCopilotアイコンが表示
04
動作確認 — はじめてのコード補完
Pythonファイルを作成し、コメントを書いてTabキーを押すと補完が表示されます。
PYTHON — hello_copilot.py 
# フィボナッチ数列を返す関数を作成する
def fibonacci(n: int) -> list[int]:
    # ↑ ここでTabを押すとCopilotが補完してくれます！
    result = [0, 1]
    while len(result) < n:
        result.append(result[-1] + result[-2])
    return result[:n]
✅
補完を受け入れる：Tab キー　　　却下する：Esc キー　　　次の候補：Alt+] / Option+]
05
Copilot Chatを使ってみる
サイドバーのCopilotアイコンをクリック、またはCtrl/Cmd+Shift+Iでチャット画面を開きます。
  • /explain — 選択したコードを日本語で説明してもらう
  • /fix — バグを自動修正してもらう
  • /tests — 単体テストを自動生成
  • /doc — ドキュメントコメントを自動生成
  • @workspace — プロジェクト全体を参照して質問
06
カスタム指示ファイルの設定（推奨）
プロジェクトルートに .github/copilot-instructions.md を作成すると、Copilotにプロジェクト固有の指示を与えられます。
.github/copilot-instructions.md 
## ビルドコマンド
- `npm run build` - プロジェクトをビルド
- `npm run test` - テストを実行
- `npm run lint:fix` - Lint自動修正

## コードスタイル
- TypeScript strict modeを使用
- 関数コンポーネントをクラスより優先
- 公開APIには必ずJSDocコメントを付与

## ワークフロー
- コミットはConventional Commits形式
- ブランチはmainから作成

効果的なプロンプト技術

AIへの指示の質がアウトプットの質を決定します。良い例と悪い例を比較しましょう

💡
プロンプトエンジニアリングの3原則： ①タスクを細分化する　②要件を具体的に書く　③入出力の例を示す

// BAD vs GOOD — プロンプト比較

❌ BAD — 漠然とした指示
ユーザー認証を作って
✅ GOOD — 具体的な指示
Next.js 14 + TypeScriptで、メールとパスワードによるJWT認証を実装して。 - bcryptでパスワードハッシュ化 - accessToken (15分) / refreshToken (7日) の2トークン方式 - エラーはカスタムエラークラスで返す - Zodでバリデーション 型定義とユニットテストも含めて
❌ BAD — コンテキスト不足
このコードのバグを直して
✅ GOOD — コンテキスト付き
@workspace の src/api/users.ts、135行目でTypeErrorが発生しています。 「Cannot read properties of undefined (reading 'id')」というエラーです。 ユーザーが存在しない場合の null チェックが漏れていると思います。 修正案と、同様のパターンが他にないか確認してください

// スラッシュコマンド チートシート

/explain
選択したコードの動作を詳細に説明します。チームへの共有や自分の理解確認に最適。
/fix
バグや問題を検出して修正案を提示。エラーメッセージと一緒に使うと精度が上がります。
/tests
Jest・Vitest等でユニットテストを自動生成。エッジケースも考慮したテストコードが得られます。
/doc
JSDoc・Docstring等のドキュメントコメントを生成。API仕様書の自動化に役立ちます。
@workspace
プロジェクト全体のファイルを参照して回答。「このプロジェクトで〇〇を実装するには？」という質問に対応。
@terminal
ターミナルの出力内容やエラーを参照して助言。シェルコマンドの問題解決に便利です。

2026年の主要機能

最新バージョンで追加された強力な機能を活用しましょう

🤖
エージェントモード NEW
自律的に複数ファイルを横断して編集・テスト実行・バグ修正まで行う高度な自動化機能。 「ユーザー認証モジュールを実装して」と指示するだけで、ファイル作成からテストまで一括実行します。
  • 複数ファイルの同時編集が可能
  • テスト実行結果を見てコードを自動修正
  • プレミアムリクエストを複数消費する点に注意
📋
プランモード（Plan Mode） 2026
コードを書く前に実装計画を作成し、承認後に実装を進める機能。Shift+Tab でトグル切替。 複雑な機能実装前に計画をレビューできるため、手戻りを大幅に削減できます。
COPILOT CHAT — Plan Mode 
# Shift+Tab でPlan Modeに切り替えた後
OAuth2認証（Google・GitHubプロバイダー）を実装して

# Copilotが以下のような計画を作成してくれます：

- [ ] NextAuth.jsのインストールと設定
- [ ] Google OAuth設定ファイル作成
- [ ] GitHubプロバイダー設定
- [ ] コールバックルート実装
- [ ] セッション管理の設定
# → 承認後に実装開始（Ctrl+Y で計画を編集可能）
🔍
コードレビューエージェント 2026
Pull Requestに対して自動でコードレビューを行うエージェント。セキュリティ問題・バグ・コードスタイルの問題を自動検出して指摘します。
  • GitHub PR上で直接レビューコメントを生成
  • GitHub Advanced Securityとの連携でセキュリティスキャン
  • コーディング規約違反の自動検出
🔌
MCPサポート（Model Context Protocol） 2026
外部ツール（Asana、Jira、Figma等）とCopilotを直接連携できる新しいプロトコル。 Copilot CLIのGitHub MCP serverでは、Copilot Spacesのツールも含まれています。
.vscode/mcp.json — MCP設定例 
{
  "servers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "${input:githubToken}"
      }
    }
  }
}
💾
Copilot Spaces（コンテキスト管理） 2026
プロジェクト固有のコンテキストを記憶・参照する機能。コードベースの知識を蓄積し、より的確な提案を生成します。
  • リポジトリを横断した知識ベースの構築
  • クロスエージェントメモリ：CLI・IDE・コードレビュー間で学習を共有
  • Enterpriseプランではカスタムファインチューニングモデルも利用可能

10のベストプラクティス

GitHub公式ドキュメントと現場エンジニアの知見を集約した実践的なガイドライン

📂 01. 関連ファイルを開き、不要なファイルを閉じる ▼
Copilotは現在エディタで開いているファイルをコンテキストとして利用します。 実装対象のファイルと関連する型定義・テストファイルだけを開くことで、より正確な補完が得られます。 逆に無関係なファイルが大量に開いているとコンテキストが汚染されます。
💡
VS Codeの「エクスプローラー → タブ管理」で不要なファイルを整理しましょう
📝 02. 具体的なコメントでコンテキストを提供する ▼
コードを書く前に、意図を説明するコメントを先に書くと、Copilotの補完精度が劇的に向上します。 「何をするか」だけでなく「なぜそうするか」「どんな制約があるか」も書くとさらに効果的です。

 * - エラー時はカスタムAPIErrorをthrow

🧩 03. タスクを小さな単位に分解する ▼
「ECサイト全体を作って」ではなく「商品一覧APIを作って」→「カート機能を追加して」→「決済フローを実装して」と 小さなステップに分解することでCopilotの精度が大幅に向上します。 複雑なタスクほどプランモードを活用して計画を立てましょう。
✅ 04. 提案コードを必ず理解してから採用する ▼
Copilotは強力なツールですが間違いを犯すことがあります。 提案されたコードを「なぜこのロジックになっているか」を自分で理解してから採用することが必須です。 特に、セキュリティに関わる部分（認証・暗号化・SQL）は細心の注意を払いましょう。
⚠️
Copilotは「コパイロット（副操縦士）」です。「オートパイロット（自動操縦）」ではありません。最終判断は常にあなたが行います。
🎯 05. ペルソナを設定してChat品質を上げる ▼
Chatに「あなたはコード品質と可読性を重視するシニアTypeScriptエンジニアです」などのペルソナを設定すると、 より専門的な視点からのレビューや提案が得られます。

🔄 06. TDD（テスト駆動開発）と組み合わせる ▼
テストを先に書き、Copilotにテストを通す実装を生成させるTDDワークフローは非常に効果的です。 テストがあることでCopilotが生成したコードの品質を自動検証できます。 /testsコマンドでテストを先生成し、その後実装を書くアプローチも有効です。
📚 07. copilot-instructions.mdを活用する ▼
プロジェクト固有のルール（コーディング規約・ライブラリ選定・命名規則）を .github/copilot-instructions.mdに記述することで、 チーム全員が一貫した品質のコードをCopilotから得られます。 指示は簡潔・具体的に書くことが重要です。長すぎると効果が薄れます。
🌐 08. コンテキストウィンドウを意識する ▼
Copilot CLIではトークン上限の95%に近づくと自動圧縮（Auto-compaction）が発動します。 長いセッションでは重要な文脈が失われることがあります。 複雑なタスクでは定期的に新しいチャットセッションを始めるか、 重要なコンテキストをcopilot-instructions.mdに記録しておきましょう。
🚀 09. WRAPフレームワークで効果的なIssueを書く ▼
GitHub Copilotエージェントが最も効果を発揮するのは、明確に記述されたIssueです。 GitHub公式が推奨するWRAPフレームワークを活用しましょう：
W — What (何を)
達成したいことを具体的に書く
R — Reason (なぜ)
背景・理由・ユーザーへの影響を書く
A — Acceptance (完了条件)
どうなれば完了か明確にする
P — Prior Context (前提)
関連コード・制約・参考情報を添付
📊 10. プレミアムリクエストの使用量を監視する ▼
Chat・エージェントモード・コードレビューはプレミアムリクエストを消費します。 上限を超えると$0.04/リクエストの追加費用が発生します。 月ごとのリセットタイミング（1日UTC深夜）を把握し、 Settings → Copilot → Usage で使用量を定期確認しましょう。
💡
通常のコード補完（インライン補完）はプレミアムリクエストを消費しません（Freeプランを除く）
07

セキュリティと注意事項

AIが生成したコードを安全に活用するための必須知識

🔐
脆弱なコードパターンを認識する
Copilotはハードコードされた認証情報、SQLインジェクション、パストラバーサルなど一般的な脆弱性をフィルタリングしますが、 すべての脆弱性を防げるわけではありません。セキュリティ関連のコードは必ず手動レビューしてください。
📜
著作権・ライセンス確認
Copilotはパブリックリポジトリのコードで学習しています。生成されたコードが既存のOSSコードに類似する可能性があります。 Business/EnterpriseプランのIP補償を活用しつつ、重要なコードは確認しましょう。
🏢
機密情報をChatに貼らない
APIキー・パスワード・個人情報などの機密データをCopilot Chatに直接貼り付けることは避けてください。 Enterpriseプランではデータ保護協定（DPA）により適切に管理されますが、基本的な慎重さが重要です。
🧪
必ずテストを実行する
Copilotが生成したコードは、既存のテスト・コードスキャン・セキュリティテストなど 通常の品質管理プロセスをすべて通過させてください。AI生成だからといって例外にしないこと。
⚙️
除外ファイルを設定する
機密性の高いファイル（.env・設定ファイル・秘密鍵）はCopilotが参照しないよう .copilotignoreまたはOrganization設定で除外できます（Business/Enterpriseプラン）。
🔄
古いAPIパターンに注意
Copilotの学習データには古いAPIやライブラリバージョンが含まれています。 生成されたコードが使用している外部ライブラリの最新バージョンと互換性があるか必ず確認しましょう。
